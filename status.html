🕹️ 專案功能(v2)

v2 全面重寫:物理核搬進 src/engine/(SI 單位、決定性),渲染改 Three.js 真 3D。遊戲、重播、全部五層測試走的都是同一條 stepGame() 路徑 —— 測到的就是玩到的。

🎮 對戰模式(/ 主頁)

  • 你 vs AI 藍=你、橘=AI,弱/中/強三難度
  • PAR-11 計分 先到 11 分、需淨勝 2 分,發球權跟著得分走
  • 死因標語 每回合結束顯示誰得分+原因(tin/出界/兩彈未接…)
  • 終局重賽 勝負畫面 + 「再來一場」按原難度或換難度

🏓 六球路 + 揮拍手感

  • 球路 drive 抽球 / lob 挑高 / drop 放小 / kill 殺球 / boast 反角(bot)/ serve 發球
  • 按鍵緩衝揮拍 按下開 8-tick 揮拍窗,進拍面就打 —— 有 timing 感不吃玄學
  • 瞄準 揮拍當下按住 ←/→ 打向那側邊線,預設打遠角
  • 解不出就降級 選的球路物理解不出 → 自動改鏟球救回,不憑空消失

⌨️ 鍵盤 + 📱 觸控

  • 移動 WASD / 方向鍵(上=往前牆)
  • 擊球鍵 空白/J=抽球 · K=挑高 · L=放小 · ;=殺球;輪到你發球按任一擊球鍵
  • 觸控 左半螢幕虛擬搖桿 + 右側四顆球路鈕(桌機自動隱藏)

🏟️ 真 3D 渲染(Three.js)

  • 標準球場 四牆 + tin / 發球線 / 界外線 + 玻璃後牆,實際 SI 尺寸
  • 程序化關節球員 軀幹/四肢/球拍,揮拍 pose、跑步擺臂擺腿動畫
  • 攝影機跟拍 後上方視角平滑跟球,深度感真實(非投影模擬)

🎬 決定性重播(/replay.html)

  • seed 即重播檔 URL 帶 seed + bot 等級 = 完整重播,同 URL 重載逐 tick bit 相同
  • 鏈式 hash 可肉眼對 HUD 顯示 FNV-1a 最終 hash,跟 blessed 值直接比對
  • 播放控制 播放/暫停、0.5–4× 倍速、時間軸拖曳

⚙️ 決定性引擎(src/engine/)

  • 60Hz 固定步進 SI 單位(公尺/秒),重力/阻力/四牆反彈
  • 決定性 lint 引擎層只准 +−×÷ sqrt imul,禁 Date/Math.random(vitest 強制)
  • mulberry32 PRNG 唯一隨機源,seed 進 seed 出
  • AI bot 預測落點跑位 + 球路決策,弱/中/強=反應與失誤率梯度

🧪 五層測試金字塔(L1–L5)

v2 的核心設計:遊戲手感沒有 oracle,所以分五層 —— 下三層(不變量/現實錨點/規則)全自動、每次 CI 跑;L4 用 bot 自打統計圈「合理範圍走廊」;L5 把人工認可的重播凍成 hash,人只需要看一次,之後由 CI 守。

L1
不變量(invariants)— 任何時刻必真
球在界內、無 NaN、能量不增、同 seed 同 hash。engine-ball-invariants(7)+ 決定性 lint engine-determinism-lint(9,靜態掃引擎層禁用 API)+ 重播一致 engine-replay(4)。
L2
現實錨點(reality anchors)— 對照物理常識
自由落體時間、反彈高度比、抽球速度數量級,錨在真實壁球數據上,防止「自洽但不像壁球」。engine-ball-anchors(5)。
L3
規則矩陣(rules matrix)— 逐條規則案例
PAR-11 計分/換發/賽點、發球合法性、tin/出界/兩彈死因歸屬。engine-rules-matrix(29)+ 擊球解算與 bot 決策 engine-shot-bot(10)。
L4
代理指標走廊(proxy metrics)— bot 自打統計守「好玩」
中 bot 自打數百回合取分佈:回合長中位數 6 / p90 17;球路使用率 serve 12.4% · drive 27.4% · lob 16.5% · kill 16.0% · drop 14.9% · boast 11.5%;制勝球分佈 drive 36.9% · drop 31.5% · boast 18.1% · kill 13.4%。六條走廊全在界內 → 動態感/多樣性/公平性有數字看門。
L5
人工賜福(golden replay blessing)— 手感的最終權威是人眼
在 /replay.html 看過認可的重播凍成 seed+ticks+hash 寫進 engine-golden-replay(3);之後任何引擎改動若動到手感 → hash 變 → CI 紅 → 必須重看重賜福。配 Playwright e2e 2 支(game.spec.ts 可玩流程、replay-viewer.spec.ts 重播一致)。
